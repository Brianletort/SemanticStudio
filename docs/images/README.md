# Documentation Images

This folder contains screenshots and diagrams used in AgentKit documentation.

## Required Screenshots

Please capture the following screenshots from a running AgentKit instance:

| Filename | Description | Recommended Size |
|----------|-------------|------------------|
| `hero-chat.png` | Main chat interface with a sample conversation | 1200×800px |
| `admin-dashboard.png` | Admin overview page with stats | 1200×800px |
| `admin-agents.png` | Domain agent list and management | 1200×800px |
| `admin-models.png` | Model configuration panel | 1200×800px |
| `admin-graph.png` | 3D knowledge graph visualization | 1200×800px |
| `admin-etl.png` | ETL jobs list and pipeline view | 1200×800px |
| `admin-observability.png` | Observability overview with stats and mode distribution | 1200×800px |
| `admin-observability-quality.png` | Quality metrics tab with trends and radar chart | 1200×800px |
| `admin-observability-users.png` | User activity tab with session drill-down | 1200×800px |
| `chat-modes.png` | Mode dropdown selector | 600×400px |
| `reasoning-pane.png` | Reasoning/trace pane expanded | 800×600px |

## Screenshot Guidelines

- **Use sample data** that looks realistic but contains no real PII
- **Use light mode** for better readability in documentation
- **Hide browser chrome** (use fullscreen or presentation mode)
- **Ensure good contrast** for accessibility
- **Optimize file size** (use PNG for UI screenshots, compress appropriately)

## Optional Diagrams

- `architecture-overview.png` - Static architecture diagram (as alternative to mermaid)
- `memory-tiers.png` - Memory system visualization
- `etl-par-loop.png` - PAR loop diagram

## Adding New Images

When adding new images:

1. Use descriptive, kebab-case filenames
2. Add to this README with description
3. Reference in documentation using relative paths: `docs/images/filename.png`
